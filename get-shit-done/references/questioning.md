# Guia de entrevista adaptativa

Este arquivo orienta a forma de perguntar durante `new-project` e `discuss-phase`.

## Principios
- perguntar em portugues do Brasil
- manter tom natural
- evitar sobrecarga de perguntas de uma vez
- adaptar profundidade conforme o projeto

## Pontos que merecem aparecer quando fizer sentido
- publico e contexto
- frontend
- backend
- banco de dados
- seguranca
- testes
- deploy
- observabilidade
- operacao
- manutencao

## Regra de ouro
O objetivo nao e despejar checklist.
O objetivo e fazer perguntas melhores e descobrir o que realmente importa para aquele projeto.
